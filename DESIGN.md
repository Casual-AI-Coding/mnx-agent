# mnx-agent 前端设计契约

## 设计目标

mnx-agent 是面向 MiniMax API 的生产力工作台。界面优先保证高密度参数配置、清晰的生成反馈和低干扰的资源复用入口。新增 UI 应融入现有控制台风格，而不是重做视觉系统。

## 视觉语言

- 布局使用现有页面的卡片式分区：`rounded-xl`、`border-border/50`、`bg-card`、轻量阴影。
- 表单区保持左侧配置、右侧结果的工作台结构；小型增强功能插入左侧配置流，不抢占主操作按钮。
- 颜色使用主题 token：`primary`、`secondary`、`accent`、`muted`、`destructive`，禁止新增硬编码品牌色。
- 图标使用 Lucide SVG，禁止使用 emoji 作为 UI 图标。

## 组件约定

- 按钮使用 `Button` 组件；主要动作使用 `default`，辅助选择动作使用 `outline` 或 `secondary`。
- 容器使用 `Card`、`CardHeader`、`CardTitle`、`CardDescription`、`CardContent`。
- 下拉选择使用现有 `Select` 组件；选择器必须提供空状态和加载状态。
- 资源引用入口命名为“资源引用”，作为生成页配置卡片之一，显示 Prompt 模板、素材和工作流模板三个来源。

## 交互状态

- 加载态必须显示明确文案，不使用空白占位。
- 无数据时显示“暂无可用资源”类空状态，不阻断原有手动输入流程。
- 应用资源后只更新对应输入字段或引用 URL，并在后续生成历史 metadata 中记录引用来源。
- 清空页面时同步清除已选择资源引用。

## 响应式与可访问性

- 移动端保持单列配置流；桌面端沿用现有生成页的双列布局。
- 所有交互控件必须有按钮语义或表单标签；资源应用按钮必须能通过文本定位。
- hover/focus 只用于说明可交互状态，不添加无意义装饰动画。

## R-021 接受约束

- 优先复用已有 `/api/materials`、`/api/templates`、`/api/workflows` 前端 API。
- 不在 R-021 中实现资源置顶、收藏排序或全新的资源管理信息架构。
- 资源使用追踪先写入生成历史和上传媒体 metadata；若后续需要跨任务统计，再增加后端引用表。
