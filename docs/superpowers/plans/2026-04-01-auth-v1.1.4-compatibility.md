# v1.1.4 功能兼容性改造计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 系统性排查并改造所有现有功能，使其与用户/角色体系兼容。

**Architecture:** 后端所有路由添加 owner_id 过滤，前端所有 API 调用添加 JWT 认证，审计日志关联用户。

**Tech Stack:** Express, PostgreSQL, React, Zustand

**Depends on:** v1.1.3 完成

**Delivers:** 所有现有功能正确处理用户隔离和权限控制

- [ ] **Step 3: GET/PATCH/DELETE /jobs/:id — 验证 owner**

```typescript
router.get('/jobs/:id', asyncHandler(async (req, res) => {
  const { id } = req.params
  const job = await dbService.getCronJobById(id)
  
  if (!job) {
    res.status(404).json({ success: false, error: '任务不存在' })
    return
  }

  // Admin/super can access any job
  const user = req.user!
  if (user.role !== 'admin' && user.role !== 'super' && job.owner_id !== user.userId) {
    res.status(403).json({ success: false, error: '无权访问此任务' })
    return
  }

  res.json({ success: true, data: job })
}))
```
